# Claudio 个人AI电台 极简架构拆解
这是Claudio个人AI电台的轻量化三层架构，核心逻辑是**前端交互入口+本地中枢调度+专项API能力底座**的协同模式，最终实现AI驱动的个性化DJ电台能力。

---

## 1. 播放器界面（前端交互层）
- **核心定位**：用户侧的直接交互入口，承担音频播放、操作控制、内容展示的核心职能
- **实现形式**：支持web app、PWA、localhost本地页面三种部署形式，当前落地版本采用PWA（渐进式Web应用）
- **核心能力**：音频播放控制、用户指令输入、流式内容展示，与后端本地服务器完成双向数据通信

## 2. 本地服务器（中枢逻辑层）
- **核心定位**：整个系统的业务中枢，是所有能力调度、逻辑编排、前后端通信的核心
- **技术栈**：基于Node.js搭建
- **核心职责**：承接前端用户请求，调度底层各专项API能力，处理Claude AI的交互与推理逻辑，完成音乐、语音、日程、天气等多能力的协同编排，是连接前端交互与底层能力的核心枢纽

## 3. 多API能力层（专项能力底座）
每个API负责单一专项事务，各司其职，协同完成全链路的AI电台能力，核心模块与对应职责如下：

| 能力模块 | 核心职责 |
| :--- | :--- |
| Claude | 系统AI大脑，负责自然语言理解、DJ播报内容生成、个性化决策、用户意图识别 |
| 网易云音乐（netease） | 音乐能力底座，提供歌曲检索、播放源获取、歌词读取、音乐推荐等核心音乐服务 |
| Fish Audio（fish） | 语音合成能力，将AI生成的DJ播报文本转为自然拟人化的人声语音 |
| 飞书（feishu） | 个人日程数据接入，读取用户日程信息，实现贴合用户行程的个性化播报 |
| OpenWeather（weather） | 天气数据接入，获取实时天气信息，融入电台播报内容 |
| UPnP | 音频投屏能力，将电台的音乐、播报内容推送到家庭音响等播放设备 |

---

### 核心协同逻辑
以Claude为AI大脑，网易云音乐负责音乐供给，Fish Audio完成语音合成，飞书、天气API补充个性化场景信息，UPnP实现家庭设备播放，通过本地Node.js服务器完成全链路调度，最终打造出能读懂用户习惯、像真人DJ一样播报的专属个人AI电台。
