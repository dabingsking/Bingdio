# Bingdio — CLI 个人 AI 音乐智能体

## 1. 项目定位

**项目路径**：E:\Bingdio\music-agent\
**名称**：Bingdio
**定位**：伴侣型 CLI 智能体，作为主 Agent（Hermes / Claude Code）忙碌时的轻量替代品，专注于音乐播放、语音播报、闲聊陪伴。

**使用场景**：
- 主 Agent 执行复杂任务时，用户想听歌/闲聊
- 每日启动自动规划歌单（时间段切歌）
- 对话中动态推荐歌单
- 天气感知自动推荐
- 心情点歌
- 语音播报（克隆音色 + DJ 口播 + 智能衔接）
- 情绪曲线跟踪

**不做什么**：
- 不做复杂任务编排（主 Agent 的职责）
- 前期不做 Web UI / PWA（复用 Claudio 的方案，后期再定）

## 4. 歌单策略

**核心思路**：蒸馏式偏好学习——用户歌单作为语料，Agent 自己分析口味，再推荐同类歌曲。

**推荐来源**：
- 每日启动：从外部库（Netease）补充 + 用户歌单
- 动态推荐：按需求灵活选择来源

**导入方式**：
- 网易云音乐歌单 ID + cookie
- 本地音乐文件夹扫描
- 手动添加

**蒸馏后可分析的维度**：
- 风格分布（后摇/电子/民谣/流行）
- 情绪倾向（治愈/宣泄/平静/躁动）
- 歌手偏好
- BPM 区间
- 语种分布
- 年代偏好
- 场景标签（夜深/工作/运动/独处）

**推荐引擎**：基于 天气 + 时间 + 心情 + 历史偏好 + 情绪曲线 的 LLM 推理组合

## 5. 情绪曲线跟踪

**数据来源**：对话中分析用户语气/情绪关键词

**处理流程**：
```
对话中记录情绪标签（时间 + 情绪值）
        ↓
日终生成情绪报告（情绪变化曲线）
        ↓
次日歌单联动（按情绪趋势调整推荐）
```

---

## 6. 智能衔接

**功能**：克隆音色 + DJ 风格过渡语，听感更自然

**示例**：
- 歌曲 A 结束 → "这首歌让我想到…" → 过渡语 → 歌曲 B 开始
- 不只是"下一首"，而是有情感的衔接

---

## 7. 交互设计

**交互形态**：对话式 + 命令式混合，Agent 可主动询问

**CLI 启动流程**：
```
CLI 启动
  → 自动获取天气 + 时间
  → 生成当日歌单（按时间段规划）
  → 克隆音色播报开场白
  → 播放第一首
```

**对话中动态推荐**：
```
用户: "最近工作有点累"
  → LLM 理解情绪 + 意图
  → 从歌单/外部库推荐
  → 自然带出推荐结果
```

## 8. 架构分层

```
bingdio CLI
├── cli/              ← 交互层（Textual TUI）
├── agent/            ← 意图理解 + 决策生成（MiniMax / MiMo）
├── tools/            ← 各专项能力
│   ├── netease/      ← 网易云音乐搜索 + 播放源
│   ├── tts/          ← 语音合成（MiMo TTS / GPT-SoVITS）
│   ├── mixer/        ← 混音（FFmpeg）
│   ├── weather/      ← 天气获取（OpenWeatherMap）
│   └── playlist/     ← 歌单管理 + 推荐引擎 + 情绪跟踪
├── memory/           ← 播放历史 + 用户偏好 + 情绪曲线（SQLite）
└── config/           ← 配置文件 + 音色配置
```
